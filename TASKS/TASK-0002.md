# TASK-0002 — Add Acceptance Runner Script

## Goal

Add a single command that runs basic repo checks (format/lint/tests) without guessing the tech stack.

## Scope (files allowed to touch)

- scripts/*
- package.json (only if it exists)
- pyproject.toml / requirements.txt (only if they exist)
- Makefile (optional)
- TASKS/TASK-0002.md
- REPORTS/AGENT_REPORT.md
- CHANGELOG.md

## Non-goals

- Do not introduce new major dependencies.
- Do not refactor existing source code.
- Do not change runtime behavior of Elysia modules.

## Implementation requirements

1. Create one entry point command:
   - Preferred: scripts/acceptance.(sh|ps1)
   - Optional: Makefile target `acceptance`

2. The script must:
   - Detect stack based on existing files:
     - If package.json exists: run npm/yarn/pnpm checks ONLY if scripts exist (e.g., lint/test/format).
     - If pyproject.toml exists: run python checks ONLY if tooling is already configured.
   - If no tooling is configured, it must PRINT a clear message and exit 0 (not failing the pipeline).

## Acceptance

1. Running the acceptance command completes successfully on a clean repo.

2. It does not error if no linter/test framework is configured.

3. It prints which checks were executed and which were skipped (and why).

4. REPORTS/AGENT_REPORT.md includes:
   - Commands added
   - Example output from one run

5. CHANGELOG.md has entry for TASK-0002.

## Rollback

Remove the newly created script/Makefile changes and revert any config edits.
