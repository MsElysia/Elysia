# TASK-0013 — Local Control Panel MVP (FastAPI)

## Goal

Provide a local-only web UI for:
- status
- review queue approve/deny
- set current task
- run acceptance

## Scope

- project_guardian/ui/* (new)
- project_guardian/review_queue.py (read-only usage)
- project_guardian/approval_store.py (read/write usage)
- CONTROL.md (read/write via UI)
- scripts/acceptance.ps1 (invoked by UI)
- tests/* (add minimal tests if framework exists)
- REPORTS/AGENT_REPORT.md
- CHANGELOG.md
- TASKS/TASK-0013.md
- CONTROL.md

## Non-goals

- No external hosting
- No auth (local-only bind to 127.0.0.1)
- No front-end framework required (plain HTML templates ok)

## Requirements

1. Add FastAPI app:
   - Bind default: 127.0.0.1:8000
   - Pages:
     - GET / : status dashboard
     - GET /reviews : list pending
     - GET /reviews/{id} : detail
     - POST /reviews/{id}/approve
     - POST /reviews/{id}/deny
     - POST /control/task : set CURRENT_TASK
     - POST /control/run-acceptance : run scripts/acceptance.ps1 and return output

2. UI Rendering
- Simple server-side HTML (Jinja2) or minimal static HTML + fetch()
- Show actions as buttons/forms

3. Safety
- Must not allow arbitrary command execution; only run acceptance script.
- Must redact sensitive context fields in UI (if any exist).

## Acceptance

- Start server and load UI locally
- Approve/deny writes to approval_store.json and updates queue status
- Set CURRENT_TASK updates CONTROL.md correctly
- Run acceptance returns output in UI
- acceptance.ps1 still passes

## Rollback

Remove ui module and revert any touched files.
