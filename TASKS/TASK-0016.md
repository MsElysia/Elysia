## Critical findings to address

1. **"review" semantics are internally inconsistent**
   - `validate_trust_for_action` returns `TrustDecision(allowed=True, decision="review", ...)`
   - Gateways treat "review" as "do not proceed, enqueue, raise TrustReviewRequiredError"
   - `allowed=True` is misleading for review decisions
   - **Fix:** Define invariant: `decision in {"deny","review"} => allowed==False`, then adjust TrustDecision construction OR document why different and ensure callers use `decision`, not `allowed`.

2. **Action namespace leak still exists**
   - `test_review_queue.py` hardcodes `"network_access"` instead of using `NETWORK_ACCESS` constant
   - **Fix:** Update `tests/test_review_queue.py` to import and use `NETWORK_ACCESS` constant.

3. **Guardrail check is weak**
   - Unknown action warning check is redundant and doesn't use `LEGACY_ACTIONS` meaningfully
   - **Fix:** Add smoke test that verifies unknown action triggers warning via mocked `memory.remember`.

## Smoke tests required (`tests/test_trust_smoke.py`)

A) **Returns TrustDecision and valid fields**
   - `decision in {"allow","deny","review"}`
   - `reason_code` non-empty
   - `message` non-empty
   - `risk_score` is None or within [0,1]

B) **Threshold behavior**
   - Set trust level explicitly
   - Verify: below required => deny, within +0.1 margin => review, above margin => allow
   - Use NETWORK_ACCESS threshold 0.7

C) **Context redaction**
   - Pass context with `{"body": "...", "token": "...", "target": "example.com"}`
   - Verify memory log only includes safe subset (filters `["sensitive","content","body"]`)
   - Document gaps if token/key/secret not filtered

D) **Unknown action guardrail triggers**
   - Mock `memory.remember` and confirm it logs warning for unknown action strings
