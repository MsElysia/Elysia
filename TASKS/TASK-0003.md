# TASK-0003 — Add Governance Invariant Tests

## Goal

Add automated checks that enforce SPEC.md invariants so agents cannot silently break governance.

## Scope (files allowed to touch)

- tests/* or __tests__/* (create if missing)
- scripts/*
- SPEC.md (only if clarification needed; prefer not)
- TASKS/TASK-0003.md
- REPORTS/AGENT_REPORT.md
- CHANGELOG.md

## Non-goals

- Do not implement full TrustEngine/MutationFlow logic.
- Do not modify existing core module runtime code unless required to expose metadata/hooks.
- Do not add external services.

## Invariants to enforce (from SPEC.md)

1. IdentityAnchor required for external actions

2. TrustEngine gates autonomy

3. MutationFlow cannot mutate core governance without explicit override

4. PromptRouter outputs deterministic + schema-valid

## Implementation requirements

Create lightweight "invariant tests" as one of:
- Unit tests (preferred if a test framework exists)
- A script-based test runner (fallback) that scans for required patterns/contracts

The tests must be conservative and non-brittle:
- If a module doesn't exist yet, test should SKIP with explanation (not fail).
- If module exists, test must assert the invariant holds.

## Acceptance

1. Invariant checks are runnable via the acceptance command from TASK-0002 (if present).

2. Each invariant check reports PASS / FAIL / SKIP with reason.

3. At least one invariant check must be "real" (not all skips) if corresponding modules exist.

4. REPORTS/AGENT_REPORT.md includes:
   - How to run invariants
   - Output sample

5. CHANGELOG.md has entry for TASK-0003.

## Rollback

Remove new tests/scripts and revert any minimal hooks added.
