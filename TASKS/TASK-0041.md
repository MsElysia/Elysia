# TASK-0041 — FileWriter Path Safety (No Traversal + Repo-Root Enforcement)

## Goal

Prevent path traversal / arbitrary file write by enforcing strict path safety in `FileWriter.write_file()`:
* reject absolute paths
* reject `..` / traversal
* enforce "must be within repo root"
* keep behavior deterministic and testable

This is a hardening task with tests. No new capabilities.

## Scope (allowed files)

* `project_guardian/file_writer.py`
* `project_guardian/core.py` (only if FileWriter needs repo-root injected; prefer minimal change)
* `tests/test_file_writer_path_safety.py` (new)
* `SPEC_MODULES/gateways.md` (update)
* `REPORTS/module_audit_gateways.md` (update)
* `TASKS/TASK-0041.md` (new)
* `CHANGELOG.md`
* `REPORTS/AGENT_REPORT.md`
* `CONTROL.md`

### Non-goals

* No new dependencies
* No changes to TrustMatrix semantics
* No UI changes

## Requirements

### 1) Define repo root resolution (deterministic)

In `FileWriter` implement a helper to determine repo root:

Preferred:
* Accept `repo_root: Path | None` in `FileWriter.__init__()`
* If None, compute from file location:
  * repo root = `Path(__file__).resolve().parents[1]` (adjust if needed so it points to the project root containing `project_guardian/`)
* Store `self.repo_root` as resolved Path

If you can avoid touching core.py by defaulting correctly, do that.

### 2) Path validation rules

Before any write:
* Input `path` can be str or Path.
* Normalize to resolved candidate:
  * `candidate = (repo_root / path).resolve()` if path is relative
* Reject if:
  * path is absolute (`Path(path).is_absolute()`)
  * any path part is `..`
  * resolved candidate is not under repo_root (`repo_root in candidate.parents` or `candidate == repo_root` is False)
* Also reject writing directly to directories (candidate endswith separator / is dir)

On rejection:
* Raise `TrustDeniedError` (preferred) with reason_code:
  * `PATH_TRAVERSAL_BLOCKED`
  * include context: original path, resolved path (safe), repo_root
  OR if you already use ValueError for mode issues, keep ValueError for mode, TrustDeniedError for safety.

### 3) Ensure gating still occurs, but only after safety checks

Order must be:
1. validate path safety
2. TrustMatrix gate (FILE_WRITE) with context including:
   * `target`: relative path from repo_root (not absolute)
   * `mode`
   * `bytes`: length of content
   * `allow_overwrite`: boolean (if applicable)
3. perform write

### 4) Atomic write behavior preserved

If FileWriter already writes atomically, preserve it.
If not, implement:
* write to temp in same directory then `os.replace`

### 5) Update docs

Update `SPEC_MODULES/gateways.md`:
* FileWriter path safety rules

Update `REPORTS/module_audit_gateways.md`:
* risk reduced (path traversal)

## Tests — `tests/test_file_writer_path_safety.py`

Use `tmp_path` as repo root by monkeypatching/injecting FileWriter(repo_root=tmp_path).

Must test:

1. Blocks absolute path
   * On Windows and POSIX, use something like:
     * POSIX: `/etc/passwd`
     * Windows: `C:\\Windows\\system32\\drivers\\etc\\hosts` (skip on non-win if needed)
   * Expect TrustDeniedError reason_code PATH_TRAVERSAL_BLOCKED

2. Blocks traversal
   * `../outside.txt`
   * `project_guardian/../outside.txt`
   * Expect blocked

3. Allows safe relative path
   * `REPORTS/test.txt` or `project_guardian/_tmp.txt`
   * Mock TrustMatrix allow
   * Assert file created under tmp_path and content matches

4. Rejects if resolved path escapes via symlink (if feasible)
   * Optional but good:
     * Create symlink inside repo pointing outside; attempt write through it; should be blocked by resolve()+parent check.
     * If symlink creation not supported, skip test.

5. Ensure gating not called on invalid path
   * Mock TrustMatrix and assert validate_trust_for_action not invoked when path invalid.
