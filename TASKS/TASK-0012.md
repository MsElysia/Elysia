# TASK-0012 — Review Queue + Approval Tokens

## Goal

When TrustDecision.decision == "review", create a durable review request instead of hard failing, while still preventing the action until approved.

## Scope

- project_guardian/trust.py
- project_guardian/external.py
- project_guardian/file_writer.py
- project_guardian/subprocess_runner.py
- project_guardian/core.py (only if needed)
- project_guardian/review_queue.py (new)
- project_guardian/approval_store.py (new)
- tests/test_invariants.py
- REPORTS/AGENT_REPORT.md
- CHANGELOG.md
- TASKS/TASK-0012.md
- CONTROL.md

## Non-goals

- No UI
- No external database (use file-based jsonl storage)

## Requirements

1. Create ReviewQueue (file-backed):
   - Append-only JSONL: /REPORTS/review_queue.jsonl (or /QUEUE/)
   - Method: enqueue(request: ReviewRequest) -> request_id
   - Method: list_pending() -> list

2. Create ApprovalStore (file-backed):
   - JSON map: request_id -> approval record (approved/denied, timestamp, approver, notes)

3. Define ReviewRequest schema:
   - request_id (uuid)
   - component, action, context
   - created_at
   - status: pending/approved/denied

4. Gateways behavior:
   - If decision == "deny": raise TrustDeniedError (as today)
   - If decision == "review":
     - enqueue ReviewRequest
     - raise TrustReviewRequiredError(request_id=..., summary=...)
   - If decision == "allow": proceed

5. Replays:
   - If a request_id is provided in context and is approved, bypass review and allow exactly that action+context match.
   - Must verify context match (prevent "approve one thing, use for another").

## Acceptance

- Tests cover:
  - review decision enqueues request and raises TrustReviewRequiredError
  - approval allows the same action/context with request_id
  - approval does NOT allow modified context
- acceptance.ps1 passes
- AGENT_REPORT includes demo showing a review request created and then approved to proceed

## Rollback

Remove new queue/store modules and revert gateway changes.
