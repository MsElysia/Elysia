## Required Features

### A) Persisted history artifacts (minimal)

When `POST /control/run-once` runs, continue writing:
* `REPORTS/run_once_last.json`

Add history retention:
* Write a copy to `REPORTS/run_once_history/`:
  * filename: `YYYYMMDD_HHMMSS_<status>_<task-or-none>.json`
  * include the same content as last.json plus timestamp

For acceptance runs:
* You already have `REPORTS/acceptance_last.json` and `.log`
  Add:
* copy acceptance_last.json to `REPORTS/acceptance_history/` with similar timestamped naming
* (optional) copy acceptance_last.log too, but cap size or truncate safely

All history writes must be atomic.

### B) History pages

Add pages:
1. `GET /history`
   * Shows:
     * recent run_once history files (latest first)
     * recent acceptance history files (latest first)
   * Each item links to detail page.

2. `GET /history/run-once/{filename}`
   * Displays parsed JSON safely (escaped)
   * Show key fields prominently (status, task_id, outcome, request_id if present)
   * Link back to history list

3. `GET /history/acceptance/{filename}`
   * Displays parsed JSON
   * Link to corresponding log if exists (view log route)

### C) Review history

Right now you show pending requests. Add "decided" view:
* `GET /reviews?status=pending|approved|denied|all`
  Implementation:
* Parse `REPORTS/review_queue.jsonl` and return latest record per request_id
* Filter by status
* Show counts by status

### D) Mutation payload browser (read-only)

Add:
1. `GET /mutations`
   * Lists `MUTATIONS/*.json` (sorted by modified time desc)
   * Show summary field + touched_paths count
   * Link to mutation detail

2. `GET /mutations/{filename}`
   * Show:
     * summary
     * touched_paths
     * per-file changes with "View Diff" links

### E) Diff viewer (read-only)

Add:
* `GET /diff?mutation=<file>&path=<changed_path>`
  Behavior:
* Load mutation payload JSON from `MUTATIONS/<file>`
* Find matching change by path
* Compare:
  * "proposed" content (from payload)
  * "current" content (from filesystem) if exists, else show "missing"
* Render unified diff using Python stdlib `difflib.unified_diff`
* Must escape output (no raw HTML injection)

Safety:
* Validate inputs:
  * mutation filename must be in MUTATIONS directory, end .json, no `..`
  * path must be one of the touched_paths in that payload (no arbitrary file read)
* Limit diff size (e.g., max 2000 lines shown) with truncation message.
