# TASK-0042 — MutationEngine Path Safety Parity (Repo-Root + No Traversal + Symlink Escape Block)

## Goal

Ensure `MutationEngine` enforces the same path safety guarantees as `FileWriter`:
* reject absolute paths
* reject `..` traversal
* enforce "must resolve within repo root"
* block symlink escape
* guarantee preflight blocks before any writes

This closes any remaining "mutation writes anywhere" hole.

## Scope (allowed files)

* `project_guardian/mutation.py`
* `project_guardian/core.py` (only if MutationEngine needs repo_root injected; prefer defaulting like FileWriter)
* `tests/test_mutation_path_safety.py` (new)
* `SPEC_MODULES/mutation.md` (update)
* `REPORTS/module_audit_mutation.md` (update)
* (Optional) `SPEC_MODULES/gateways.md` (only if it references mutation path rules; keep minimal)
* `TASKS/TASK-0042.md` (new)
* `CHANGELOG.md`
* `REPORTS/AGENT_REPORT.md`
* `CONTROL.md`

### Non-goals

* No patch/diff support
* No redesign of review/replay semantics
* No changes to TrustMatrix thresholds
* No UI changes

## Requirements

### 1) Repo root resolution (deterministic)

Add `repo_root: Path | None = None` to `MutationEngine.__init__()`:
* If None, compute project root similarly to FileWriter:
  * `Path(__file__).resolve().parent.parent`
* Store `self.repo_root` as resolved Path

Do not require core.py changes if you can default correctly.

### 2) Centralize path normalization/validation

Create helper(s) inside mutation.py:
* `_validate_and_resolve_path(rel_path: str) -> tuple[Path, str]`
  * Input must be relative path
  * Reject if absolute
  * Reject if any part == ".."
  * Resolve: `(repo_root / rel_path).resolve()`
  * Enforce `resolved.relative_to(repo_root)` succeeds
  * Return (resolved_path, normalized_rel_path_from_repo_root)

If invalid:
* Raise `MutationDeniedError` with reason_code:
  * `PATH_TRAVERSAL_BLOCKED`
  * Include context details (safe):
    * `path`, `repo_root`, `resolved` (string)

### 3) Apply validation to *all* touched paths

In any code path that applies mutations:
* Validate every `touched_path` and every `changes[].path` using the helper
* Ensure `touched_paths` set equality check is done against normalized paths (consistent)

### 4) Preflight must block before writes

Your current preflight in APPLY_MUTATION should:
* validate all paths using the new helper
* if any invalid → deny before any backups/writes occur
* this must hold both for governance and non-governance mutations

### 5) Prevent directory writes

If a resolved path points to an existing directory (or ends with separator), deny with reason_code:
* `PATH_IS_DIRECTORY`

### 6) Tests — `tests/test_mutation_path_safety.py`

Use `tmp_path` as repo_root by injecting `MutationEngine(repo_root=tmp_path)`.

Must test:

1. Blocks absolute path in payload
   * changes path `/etc/passwd` (POSIX) or `C:\...` (Windows)
   * Expect MutationDeniedError reason_code PATH_TRAVERSAL_BLOCKED

2. Blocks traversal
   * `../outside.txt`
   * `a/../outside.txt`
   * Expect blocked

3. Blocks symlink escape (if supported)
   * Create outside dir + file
   * Create symlink inside repo pointing to outside
   * Attempt mutation write through symlink path
   * Expect blocked (PATH_TRAVERSAL_BLOCKED)

4. Denies before writing anything
   * Prepare payload with 2 files: first valid, second invalid traversal
   * Run apply/preflight path
   * Assert valid file remains unchanged AND no backups created for it

5. Allows safe path
   * Valid file path under repo_root
   * TrustMatrix allow (or bypass if non-governance path doesn't require governance)
   * Apply mutation and verify content changed + backups exist

### 7) Update docs

Update `SPEC_MODULES/mutation.md`:

Update `REPORTS/module_audit_mutation.md`:
