# TASK-0050 — READ_ONLY_ANALYSIS Task Type (First Real Job)

## Purpose (why this exists)

Give Elysia her **first real job** that produces value **without risk**.

This task introduces a **read-only analytical capability** that:

* exercises the full task engine
* uses governance + TrustMatrix correctly
* produces durable, explainable artifacts
* performs **zero mutations, zero subprocess execution**

This is the bridge from "secure kernel" → "useful system".

## High-level goal

Add a new **task type**:

```
TASK_TYPE: READ_ONLY_ANALYSIS
```

That:

* reads files and/or URLs
* analyzes them
* writes a structured report to `REPORTS/`
* never mutates the repo
* never launches subprocesses
* never writes outside REPORTS

## Non-goals (strict)

* ❌ No mutations
* ❌ No FileWriter usage
* ❌ No SubprocessRunner usage
* ❌ No network writes
* ❌ No task chaining
* ❌ No UI changes

This task **must not** weaken invariants.

## Task contract format

A task file under `TASKS/` must contain:

```
TASK_TYPE: READ_ONLY_ANALYSIS
ANALYSIS_KIND: <one of: REPO_SUMMARY | FILE_SET | URL_RESEARCH>
INPUTS:
  - type: repo | file | url
    value: <string>
OUTPUT_REPORT: REPORTS/<name>.json
```

Rules:

* Exactly **one** ANALYSIS_KIND
* OUTPUT_REPORT must:
  * be under REPORTS/
  * end in `.json`
  * not contain `..`
* INPUTS must be non-empty

Invalid contracts must return `{status:"error", code:"TASK_CONTRACT_INVALID"}`

## Behavior requirements

### 1. Core routing

Extend `GuardianCore.run_once()`:

* When `TASK_TYPE == READ_ONLY_ANALYSIS`:
  * Validate contract
  * Call `_execute_read_only_analysis()`
  * Return structured result dict only

### 2. Analysis engine (new module)

Create `AnalysisEngine` with method:

```python
def run(kind: str, inputs: list[dict]) -> dict:
```

Responsibilities:

* Read-only filesystem access
* Optional WebReader GET requests (subject to TrustMatrix)
* No writes except returning data to Core
* Deterministic output

Initial supported kinds:

#### A) REPO_SUMMARY

* Walk repo (excluding venv, **pycache**, REPORTS, TASKS)
* Return:
  * file counts by extension
  * total LOC (rough count)
  * top-level directories
  * timestamp

#### B) FILE_SET

* Read listed files
* Return:
  * filename
  * size
  * sha256
  * first N lines (e.g., 50, configurable constant)

#### C) URL_RESEARCH

* Use WebReader.fetch()
* Extract:
  * URL
  * HTTP status
  * content length
  * first N chars
* Subject to SSRF rules and TrustMatrix
* Review/deny must halt with **no report written**

## Output rules (critical)

* Core writes output report **only if analysis completes**
* Report must be JSON with:
  * metadata (task_id, analysis_kind, inputs, timestamp)
  * results
* On deny/review:
  * **no report file created**
  * review allowed to enqueue ReviewQueue entry only

## Trust & safety rules

* Filesystem reads are allowed without TrustMatrix
* Network reads:
  * must go through WebReader
  * subject to TrustMatrix (deny/review/allow)
* Review outcome:
  * enqueue review
  * return `{status:"needs_review"}`
  * no report written
