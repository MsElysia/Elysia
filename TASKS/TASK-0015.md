# TASK-0015 — Control Plane Hardening

## Goal

Eliminate crash/race footguns and normalize governance semantics. No new UI features beyond what's required for safety.

## Scope (allowed files)

* `project_guardian/ui/app.py`
* `project_guardian/review_queue.py`
* `project_guardian/approval_store.py`
* `project_guardian/external.py`
* `project_guardian/file_writer.py`
* `project_guardian/subprocess_runner.py`
* `project_guardian/trust.py` (only if needed for action constants)
* `tests/test_invariants.py` (only to align with changes)
* `scripts/acceptance.ps1` (only if artifact assertion missing)
* `CHANGELOG.md`
* `REPORTS/AGENT_REPORT.md`
* `TASKS/TASK-0015.md`
* `CONTROL.md`

## Non-goals

- No new UI features
- No new dependencies
- No file locking library (use atomic replace + single-writer assumption)

## Requirements

1. **Atomic writes everywhere state matters**
   * Any write to `CONTROL.md`, approval store JSON, and UI-updated state must be atomic:
     * write to `*.tmp`
     * `os.replace(tmp, target)`
   * Do not leave partial files on crash.

2. **ReviewQueue must return latest state**
   * `ReviewQueue.get_request(request_id)` must return the **latest** matching record (last occurrence in JSONL).
   * Any status lookup should prefer latest appended update semantics.

3. **ApprovalStore must be race-safe**
   * Approval store writes must be atomic (tmp + replace).
   * If feasible without deps, use a minimal file lock; otherwise document "single-writer" assumption explicitly in code comments and keep atomic replace.

4. **UI must not mutate request context**
   * In `ui/app.py`, do not overwrite `req.context` when redacting.
   * Redaction should operate on a copied dict for display purposes.

5. **Normalize action strings**
   * Introduce a single source of truth for action names (constants), e.g.:
     * `NETWORK_ACCESS`
     * `FILE_WRITE`
     * `SUBPROCESS_EXECUTION`
     * `GOVERNANCE_MUTATION`
   * Update gateways to use constants consistently (avoid `file_write` vs `file_operation` mismatch).
   * Update any TrustMatrix rules/tests to match.

6. **Acceptance artifact assertion**
   * If `scripts/acceptance.ps1` does not explicitly fail when `REPORTS/acceptance_last.json` is missing after a run, add that assertion.

## Acceptance

* `.\scripts\acceptance.ps1` passes.
* Invariants pass.
* Demonstrate:
  * `CONTROL.md` update is atomic (code-level proof + comment)
  * `get_request()` returns latest record (unit test or proof snippet)
  * approval store writes are atomic
* Update `CHANGELOG.md` with TASK-0015 entry.
* Update `REPORTS/AGENT_REPORT.md` with:
  * files changed
  * commands run + outputs
  * tests run + results
  * any remaining risks/assumptions (especially around file locking)

## Rollback

Revert atomic write changes and action constant changes.
