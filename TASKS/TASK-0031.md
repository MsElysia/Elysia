# TASK-0031 — Module Completeness Matrix (Inventory + Gaps + Next Task Contracts)

## Goal

Create an objective "completeness matrix" for the codebase: inventory all modules, determine whether each has **spec**, **audit**, **tests**, **core wiring**, and **bypass compliance**, then generate:

1. A single report: `REPORTS/module_completeness_matrix.md`
2. A machine-readable file: `REPORTS/module_completeness_matrix.json`
3. A set of **auto-generated task contracts** for gaps (TASK-0032+, one per gap category or per module)

This is analysis + reporting only. No refactors unless required to read metadata.

## Scope (allowed files)

* New script:
  * `scripts/module_matrix.py` (new)
* Outputs:
  * `REPORTS/module_completeness_matrix.md` (new)
  * `REPORTS/module_completeness_matrix.json` (new)
* Optional helper:
  * `project_guardian/ui/app.py` (read-only; do not change UI in this task)
* Tests:
  * `tests/test_module_matrix.py` (new, minimal)
* Task contracts:
  * `TASKS/TASK-0031.md` (new)
  * `TASKS/TASK-0032-*.md` (new, generated by script or created by agent)
* Docs:
  * `SPEC_MODULES/` (read-only)
  * `REPORTS/` (read-only except new outputs)
* `CHANGELOG.md`
* `REPORTS/AGENT_REPORT.md`
* `CONTROL.md`

## Non-goals

* No feature development
* No adding specs for missing modules (that comes in follow-on tasks)
* No changing governance logic
* No new dependencies (stdlib only)

## Definitions (what to measure)

For each Python module file under `project_guardian/` (excluding `project_guardian/ui/templates/**`, `__init__.py` optional inclusion), compute:

### Fields

* `module`: dotted name (e.g., `project_guardian.trust`)
* `path`: relative path
* `spec_exists`: `SPEC_MODULES/<name>.md` exists (mapping rules below)
* `audit_exists`: `REPORTS/module_audit_<name>.md` exists (mapping rules below)
* `tests_exist`: any test file references module OR module has dedicated smoke test (heuristics below)
* `wired_in_core`: referenced/imported/instantiated in `project_guardian/core.py` (AST-based)
* `bypass_clean`: no direct external action calls except via allowlisted gateways (reuse/approximate existing invariant logic, but script-only)
* `status`: ✅ complete / ⚠️ partial / ❌ missing
* `notes`: short list of detected gaps

### Mapping rules

Module shortname:
* `project_guardian/trust.py` → `trust`
* `project_guardian/review_queue.py` → `review_queue`
* `project_guardian/subprocess_runner.py` → `subprocess_runner`
* `project_guardian/ui/app.py` → `ui` (special-case: spec file is `SPEC_MODULES/ui.md`, audit is `REPORTS/module_audit_ui.md`)

Spec mapping:
* default: `SPEC_MODULES/<shortname>.md`
* ui: `SPEC_MODULES/ui.md`

Audit mapping:
* default: `REPORTS/module_audit_<shortname>.md`
* ui: `REPORTS/module_audit_ui.md`

## Heuristics (must be deterministic)

### A) Detect tests_exist (reasonable heuristic)

Set `tests_exist=True` if:
* any file under `tests/` contains the shortname or dotted module name string, OR
* there is a dedicated smoke test file with name containing the shortname (e.g., `test_trust_smoke.py`)

This is approximate but deterministic.

### B) Detect wired_in_core (AST-based)

Parse `project_guardian/core.py` and detect:
* `import project_guardian.<x>` or `from project_guardian import <x>`
* attribute construction like `<x>.ClassName(` is a plus but not required
  Mark wired if imported OR referenced.

### C) Detect bypass_clean (script-level approximation)

Parse module AST and flag if it contains:
* network libs: `requests`, `httpx`, `urllib`, `aiohttp`, `websocket`, `playwright`
* subprocess: `subprocess`, `os.system`, `os.popen`
* file writes: `open(..., "w"/"a"/"wb"/"ab")`, `Path.write_text`, `Path.write_bytes`, `shutil.move/copy`, `os.replace`

AND the module is not one of the gateway modules that are allowed to do that:

* Allowed external-action surfaces:
  * `project_guardian/external.py` (network gateway)
  * `project_guardian/file_writer.py` (file write gateway)
  * `project_guardian/subprocess_runner.py` (subprocess gateway)
  * `project_guardian/mutation.py` (file writes for mutation application are allowed but should be noted)
  * `project_guardian/ui/app.py` (allowed to write REPORTS/TASKS/MUTATIONS; note as "UI writes")

For anything else, mark bypass_clean=False and include exact file+line findings.

(Do NOT attempt to perfectly mirror invariant tests; this is a reporting tool.)

## Outputs

### 1) `REPORTS/module_completeness_matrix.json`

An array of module records with the fields above.

### 2) `REPORTS/module_completeness_matrix.md`

Human-readable:
* Summary counts (complete/partial/missing)
* Table sorted by worst-first (❌ then ⚠️ then ✅)
* For each non-complete module, list gaps and recommended next task file name

### 3) Generate follow-on TASK contracts

Create at least:
* `TASKS/TASK-0032-SPECS.md` (add missing specs)
* `TASKS/TASK-0033-TESTS.md` (add missing smoke tests)
* `TASKS/TASK-0034-AUDITS.md` (write missing audit reports)
* `TASKS/TASK-0035-WIRING.md` (core wiring decisions: wire or document on-demand)
* `TASKS/TASK-0036-BYPASS.md` (resolve any bypass findings)

Each contract should include:
* scope, non-goals
* list of modules affected
* acceptance criteria
* "do not redesign architecture" rule

If there are no gaps in a category, still create the task file but mark it "NO-OP (no gaps found)" and keep it short.

## Script UX

`scripts/module_matrix.py` should run with:

```bash
python scripts/module_matrix.py
```

and print:
* where it wrote outputs
* counts summary

Exit code:
* 0 always (reporting tool)

## Tests

`tests/test_module_matrix.py` should:
* run the script main function (import) against a temp fake structure OR minimal real calls
* at minimum, assert the output JSON file is created and contains expected keys for known modules (trust, core, ui)

Keep tests small and stable.
