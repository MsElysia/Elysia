# TASK-0004 — Enforce Invariants as a Gate + Strengthen Checks

## Goal

Make invariant checks fail the pipeline on violations, and upgrade invariant tests from "presence checks" to "behavioral checks".

## Scope (files allowed to touch)

- scripts/acceptance.ps1
- tests/test_invariants.py
- REPORTS/AGENT_REPORT.md
- CHANGELOG.md
- CONTROL.md
- TASKS/TASK-0004.md

## Non-goals

- No refactors of core modules unless required to expose a minimal hook.
- No new dependencies.

## Requirements

1. acceptance.ps1 must:
   - Exit 0 if no tooling exists.
   - Exit non-zero if pytest is run and any test fails.
   - Print PASS/FAIL/SKIP summary.

2. test_invariants.py must enforce at least TWO behavioral invariants:

   A) Trust gate:
      - Identify at least one function/method in code that triggers an "external action" (web, filesystem write outside task scope, finance).
      - Assert that it calls a TrustEngine/TrustMatrix gate method before action.

   B) Mutation protection:
      - Identify governance paths (e.g., IdentityAnchor, TrustEngine, MutationFlow, PromptRouter, ElysiaLoop-Core).
      - Assert MutationFlow rejects modifications to those paths unless an explicit override flag/config is present.

3. PromptRouter should not be a SKIP if router exists:
   - If router.py exists, add a determinism test:
     - Same input -> same output (string or dict equality).
     - If nondeterminism is intentional, require explicit seed or mode flag.

## Acceptance

- Running scripts/acceptance.ps1:
  - returns non-zero when you intentionally break an invariant (add a test-only mock breach)
  - returns zero when all invariants pass
- CONTROL.md set to CURRENT_TASK: NONE after completion
- REPORTS/AGENT_REPORT.md includes proof: command outputs + exit codes

## Rollback

Revert acceptance.ps1 and test_invariants.py changes.
