## Requirements

## 1) Upgrade WebReader to support POST/PUT with JSON

Currently WebReader supports GET returning text. Add a method and keep old `fetch()` intact.

### Add in `external.py`
* `WebReader.request_json(...)` (name flexible but must be explicit)

Signature requirements (minimum):
* `method: Literal["GET","POST","PUT","PATCH","DELETE"]` (support at least POST/PUT)
* `url: str`
* `json_body: dict | list | None`
* `headers: dict | None` (allowlist key names; must redact in logs)
* `timeout_s: int` default 30
* `caller_identity: str | None`
* `task_id: str | None`
* `request_id: str | None` (for review replay)

Return:
* `dict` with keys:
  * `status_code: int`
  * `json: Any | None`
  * `text: str | None`
  * `headers: dict` (optional, may omit)

Behavior:
* Always gate using TrustMatrix action `NETWORK_ACCESS` with context including:
  * target domain (not full URL)
  * method
  * has_body (bool)
  * content_type (e.g., "json")
  * caller_identity, task_id
* For `decision="review"`: enqueue and raise TrustReviewRequiredError
* For deny: raise TrustDeniedError
* Replay: if approved request_id with matching context → bypass review and proceed

Implementation detail:
* Use stdlib `urllib.request` to avoid deps (or `requests` only if already in project; prefer stdlib)
* Parse JSON response if `Content-Type` indicates JSON or if response body starts with `{`/`[` (best-effort)
* Redact headers before logging (Authorization, token, key, secret, etc.)

### Update `SPEC_MODULES/gateways.md`
Document request_json contract and logging/redaction rules.

## 2) Upgrade SubprocessRunner to support non-blocking / async strategy

The "slave_deployment" justification was "SubprocessRunner is synchronous". Fix this without adding real async complexity.

Add to `SubprocessRunner`:
* `run_command_async(...)` **or** `run_command(..., background: bool=False)`

If `background=True`:
* Use `subprocess.Popen` but still gated under `SUBPROCESS_EXECUTION`
* Return a structured object:
  * `pid`
  * `started: bool`
  * `command`
* No stdout streaming required
* Must still enforce `shell=False`
* Must still enforce timeout for non-background runs
* Must still support review/replay like current run_command
* Consider deny list: if background, require higher trust requirement in TrustMatrix (same action is okay, but add context `background=True`)

Update specs accordingly.

## 3) Refactor the 5 modules to use gateways

Replace direct network/subprocess/file-write calls.

### A) `gumroad_client.py`
* All HTTP calls route through `WebReader.request_json(method=..., json_body=...)`
* Ensure Authorization header is passed but never logged unredacted
* If it previously used requests/httpx, remove those imports
* If it needs form encoding, convert to JSON or implement minimal form encoding in request_json (only if needed)

### B) `ai_tool_registry_engine.py`
* Route POST JSON calls through `WebReader.request_json`
* Remove direct HTTP library usage

### C) `webscout_agent.py`
* If it uses async HTTP libs, convert to sync using `WebReader.request_json` / `fetch`
* If concurrency is required, do it sequentially for now (governance > performance)

### D) `slave_deployment.py`
* Route subprocess calls through `SubprocessRunner` using background mode if it truly needs async
* Otherwise, keep it synchronous but through gateway

### E) `metacoder.py`
* Route file writes through:
  * `MutationEngine` (preferred if it's performing code mutations), OR
  * `FileWriter.write_file()` for non-mutation artifacts
* Route any subprocess test runs through `SubprocessRunner`
* Remove direct `shutil.copy/move` backup behavior unless it's inside MutationEngine (which already handles backups/governance)
