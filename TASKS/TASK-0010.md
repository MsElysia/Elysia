# TASK-0010 — AST-based Bypass Detection + Scoped Gateway Allowlist

## Goal

Make bypass detection robust (AST-based) and prevent dumping ungated external actions into "approved modules."

## Scope

- tests/test_invariants.py
- REPORTS/AGENT_REPORT.md
- CHANGELOG.md
- TASKS/TASK-0010.md
- CONTROL.md

## Non-goals

- No runtime code changes
- No third-party deps

## Requirements

1. Replace string/regex scanning with AST parsing for .py files (excluding tests/ and scripts/ if desired).

2. Detect:
   - Network: imports/calls related to requests/httpx/aiohttp/urllib/websockets/playwright
   - Filesystem writes: open/write_text/write_bytes/shutil.move/copy/os.remove/rename
   - Subprocess: subprocess/os.system/os.popen

3. Allowlist must be scoped to gateway functions, not files:
   - e.g., allow network calls only inside WebReader.fetch (or other explicit gateways)
   - allow filesystem writes only inside a defined FileWriter gateway (if exists) or deny
   - allow subprocess only inside a defined gateway or deny

4. On violation: fail with file:line + symbol.

## Acceptance

- Test reports violations with precise location
- acceptance.ps1 fails when a bypass is introduced
- No false positives in current repo state

## Rollback

Revert invariant changes.
