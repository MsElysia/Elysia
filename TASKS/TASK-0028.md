# TASK-0028 — History Retention Policy (Prune Old Run/Acceptance History + Cap Logs)

## Goal

Prevent silent disk growth by adding a **retention policy** for:
* `REPORTS/run_once_history/`
* `REPORTS/acceptance_history/`

Keep "last" artifacts untouched. Prune oldest files beyond configured limits. Cap acceptance log copies.

## Scope (allowed files)

* `project_guardian/ui/app.py`
* `project_guardian/ui/templates/history.html` (update only if showing retention info)
* `tests/test_ui_retention.py` (new)
* `SPEC_MODULES/ui.md` (update)
* `REPORTS/module_audit_ui.md` (update)
* `TASKS/TASK-0028.md` (new)
* `CHANGELOG.md`
* `REPORTS/AGENT_REPORT.md`
* `CONTROL.md`

## Non-goals

* No schedulers/background jobs
* No new dependencies
* No deleting `REPORTS/*_last.*`
* No changes to governance or task execution

## Retention requirements

### 1) File-count retention (primary)

After writing a new history artifact, prune to:
* Run once history: keep newest **MAX_RUN_ONCE_HISTORY = 200**
* Acceptance history: keep newest **MAX_ACCEPTANCE_HISTORY = 200**

Pruning rule:
* Sort by modified time (mtime) descending
* Delete files beyond max count (oldest first)

### 2) Optional age retention (secondary, if easy)

If simple, also prune files older than:
* **MAX_HISTORY_DAYS = 30**

Do this only if it's easy and reliable. If not, skip and document.

### 3) Log copy caps (acceptance)

You currently copy acceptance logs optionally.
Enforce:
* Only copy `acceptance_last.log` if size <= **MAX_LOG_BYTES = 1_000_000** (1MB)
* If larger, do not copy; instead write a small marker file in acceptance_history (or store a flag in JSON) indicating "log too large, not copied".

Do not truncate logs (truncation can mislead). Either copy full or skip with explicit marker.

### 4) Safety constraints

* Never prune outside the history directories.
* Only delete files that match expected naming patterns:
  * `run_once_history`: `*.json`
  * `acceptance_history`: `*.json` and optional `*.log`
* Do not delete directories.
* All pruning must be best-effort: failures should not crash the UI.

### 5) Surface retention info in UI (optional)

If trivial:
* Show "Retention: last 200 entries" on `/history` page.

## Implementation requirements

Add helper(s) in `project_guardian/ui/app.py`:
* `_ensure_dir(path)`
* `_prune_history_dir(path, max_files, max_days=None, allowed_suffixes=...)`

Call pruning:
* after writing `run_once_history` artifact
* after writing `acceptance_history` artifact (and log, if copied)
