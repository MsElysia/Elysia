## Required checks

### A) ReviewQueue (JSONL append-only)

1. **Append-only integrity**
   - Enqueue 3 requests, verify JSONL grows and earlier lines remain unchanged.

2. **Latest-state correctness**
   - Enqueue request -> update_status approved -> update_status denied (simulate bad sequence)
   - `get_request()` must reflect the **latest** state
   - **Decision required:** Should status transitions be monotonic (pending → approved/denied only)?
   - If monotonic, enforce in `update_status()` and test it.

3. **Restart tolerance**
   - Instantiate ReviewQueue, enqueue, re-instantiate, list_pending returns correct pending set.

4. **Corruption handling**
   - If a line is invalid JSON, queue should skip with a warning, not crash the system.

### B) ApprovalStore (JSON map + context hash)

1. **Atomic write behavior**
   - Approve one id, reload, verify persisted.

2. **Context-match strictness**
   - Approve with context A
   - `is_approved(request_id, context A)` => True
   - `is_approved(request_id, context B)` => False (must not allow partial match)

3. **Deterministic hashing**
   - Context dict ordering must not change hash results (ensure sort_keys=True).

4. **Replay attack simulation**
   - Approve request_id for NETWORK_ACCESS target=example.com
   - Attempt replay with target=malicious.com => must fail with context mismatch
