## Required changes

### 1) Add preflight to APPLY_MUTATION execution

In Core's `APPLY_MUTATION` handler (the logic you added in TASK-0023), implement a **preflight phase** that guarantees **no file writes occur** unless the entire mutation batch is allowed.

#### Preflight requirements

Given the parsed payload:
* Compute the full set of `touched_paths` from `changes[].path` (sorted).
* Validate all paths are safe (no `..`, no absolute, within repo root) BEFORE anything else.

Then:

**A) If `ALLOW_GOVERNANCE_MUTATION=false`**
* If any touched path is protected (CONTROL.md, SPEC.md, TASKS/, REPORTS/, scripts/, tests/, core modules, etc. as MutationEngine defines):
  * Return `{status:"denied", code:"MUTATION_DENIED", detail:"protected_path_without_override"}`
  * Do not call MutationEngine, do not write files.

**B) If `ALLOW_GOVERNANCE_MUTATION=true`**
* Consult TrustMatrix **once** for `GOVERNANCE_MUTATION` with context:
```python
{
  "touched_paths": sorted_paths,
  "override_flag": True,
  "caller": "GuardianCore",
  "task_id": task_id
}
```
* If decision == `"deny"`:
  * Return denied (no writes)
* If decision == `"review"`:
  * Enqueue review request (same pattern as MutationEngine/gateways)
  * Return `{status:"needs_review", request_id, outcome:"mutation_review_required"}`
  * No writes
* If decision == `"allow"`:
  * Proceed to apply

**Important:** Once preflight passes, apply the entire batch. No "apply file 1 then discover review on file 2."

#### Replay handling

If `REQUEST_ID` is provided:
* Check ApprovalStore for approval with the **same context hash** (sorted touched_paths, override_flag, caller, task_id)
* If approved, bypass the TrustMatrix review decision and proceed.

## 2) End-to-end tests: `tests/test_e2e_workflow.py` (deterministic)

Use `tmp_path` and monkeypatch Core paths (CONTROL/TASKS/MUTATIONS/REPORTS) to avoid touching the real repo.

### Test 1: Review → approve → replay → success (single file)

* Create:
  * CONTROL.md pointing to TASK-XXXX
  * TASK file with APPLY_MUTATION + ALLOW_GOVERNANCE_MUTATION:true
  * MUTATIONS payload touching a safe temp file under a temp "repo root"
* Force TrustMatrix decision = review initially
* Run `core.run_once()`:
  * Assert status=="needs_review"
  * Assert review queue contains entry
  * Assert target file is **unchanged**
* Approve request_id with matching context in ApprovalStore
* Add `REQUEST_ID` to task contract (or create a second task file)
* Run `core.run_once()` again:
  * Assert status=="ok"
  * Assert file content changed
  * Assert backups exist / MutationResult fields present

### Test 2: Preflight prevents partial apply (two files, second would trigger review/deny)

* Payload includes two changes:
  * change A: safe file
  * change B: triggers review/deny (simulate by forcing TrustMatrix decision=review)
* Run `core.run_once()`:
  * Assert status=="needs_review" (or denied)
  * Assert **neither file changed** (A must remain unchanged)
    This proves preflight prevents partial apply.

### Test 3: Protected path without override denied without writes

* Payload touches CONTROL.md or SPEC.md
* ALLOW_GOVERNANCE_MUTATION:false
* Run `core.run_once()`:
  * Assert status=="denied"
  * Assert no files written/changed
