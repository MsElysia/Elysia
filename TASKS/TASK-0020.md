# TASK-0020 — GuardianCore / Core Loop Audit (Completeness + Functionality)

## Goal

Audit **GuardianCore (core loop/orchestrator)** for completeness and correct integration with:
* TrustMatrix (TrustDecision semantics)
* ReviewQueue + ApprovalStore (review → approve → replay)
* Gateways (WebReader/FileWriter/SubprocessRunner)
* MutationEngine (explicit exceptions + MutationResult)

This task must produce: a contract spec, smoke tests, and an audit report. No feature work beyond a minimal `run_once()` wrapper if absolutely required.

## Requirements

### 1) Provide a deterministic single-step entrypoint

Core must have a **single-iteration** method:
* preferred existing: `run_once()`, `tick()`, `step()`
* if missing: add a minimal wrapper method `run_once()` that executes one cycle without infinite loops

**Rule:** wrapper must not change architecture; it only exposes one deterministic iteration for testing.

### 2) Smoke tests: `tests/test_core_smoke.py`

Add deterministic tests (no network, no real subprocess, no uncontrolled file writes).

#### A) Construction wiring test

Instantiate Core in a temp environment with:
* real TrustMatrix (or a stub that returns TrustDecision)
* ReviewQueue + ApprovalStore pointing to temp files
* Gateways injected (or ensure Core constructs them correctly)
* MutationEngine injected/constructed

Assert:
* `core.trust` exists and is TrustMatrix
* Gateways exist and reference the same TrustMatrix instance
* MutationEngine exists and references TrustMatrix
* No module is constructed without TrustMatrix

#### B) Review propagation test

Force a review decision on a safe operation (mock TrustMatrix to return `decision="review"`).
Trigger the Core path that causes that gate to be evaluated.

Assert:
* Core surfaces the review outcome explicitly (does not swallow it)
* ReviewQueue receives an entry
* A `TrustReviewRequiredError` / `MutationReviewRequiredError` propagates OR Core returns a structured "needs_review" result (choose one and document in spec)

#### C) Deny propagation test

Force `decision="deny"`.

Assert:
* Core does not proceed
* Denial is explicit and includes reason_code

#### D) No direct external actions in Core

Patch/spy at runtime:
* `requests.request` (or common network primitive)
* `subprocess.run`
* `builtins.open` for write modes

Run a Core iteration path (run_once) that should NOT require external actions.

Assert none of those primitives were called by Core directly.
(Allow gateways to call them if invoked intentionally; Core must not.)

#### E) Mutation integration test (governance)

Attempt a governance mutation via Core using MutationEngine path:
* without override: expect `MutationDeniedError`
* with override: force TrustDecision to review → expect `MutationReviewRequiredError` and queued request
* approve + replay: expect success with MutationResult

If Core has no entrypoint to propose/apply mutation, document that as gap #1 and add a minimal test-only hook ONLY if necessary.
