# TASK-0030 — Enforce Local-Only UI Access (Loopback Guard + Misbind Warning)

## Goal

Make the control panel **self-defending** against accidental exposure by enforcing local-only access at the application layer (not just host binding).

## Scope (allowed files)

* `project_guardian/ui/app.py`
* Templates:
  * `project_guardian/ui/templates/dashboard.html` (update)
  * `project_guardian/ui/templates/error.html` (update if needed)
* `scripts/start_control_panel.ps1` (update)
* `tests/test_ui_local_only.py` (new)
* `SPEC_MODULES/ui.md` (update)
* `REPORTS/module_audit_ui.md` (update)
* `TASKS/TASK-0030.md` (new)
* `CHANGELOG.md`
* `REPORTS/AGENT_REPORT.md`
* `CONTROL.md`

## Non-goals

* No full authentication system
* No reverse proxy support
* No new dependencies
* No changes to governance/task semantics

## Requirements

### 1) Add a loopback-only request guard (hard block)

Implement a FastAPI middleware (or dependency) that rejects any request where the client IP is not loopback.

Allowed:
* `127.0.0.1`
* `::1`

Denied:
* anything else

Response on deny:
* HTTP 403
* Render `error.html` with message: "Local-only UI: remote access denied."

Important:
* Consider `X-Forwarded-For` **untrusted**. Do NOT allow it to override client host.
* Use `request.client.host` only.

### 2) Add a misbind warning banner (soft signal)

If uvicorn binds to non-loopback (or app detects `host != 127.0.0.1` is impossible to know reliably from inside app), we can still signal risk:

Implement a banner in dashboard that displays:
* "Local-only UI is enforced. Ensure server is bound to 127.0.0.1."

Additionally, expose in `/api/status` (if present) a field:
* `"local_only_enforced": true`

If you can reliably detect binding host from env var, allow:
* `UI_BIND_HOST` env var read by app; if set and not loopback, show red warning.

### 3) Update start script to force loopback binding (belt + suspenders)

In `scripts/start_control_panel.ps1` ensure uvicorn starts with:
* `--host 127.0.0.1`
* `--port 8000`

If the script already does, add an explicit warning if user edits it or passes different host (best-effort).

### 4) Tests: `tests/test_ui_local_only.py`

Using TestClient:
* Test default requests succeed (TestClient will appear as local; should pass).
* Simulate remote host by overriding `request.client.host` is not straightforward in TestClient; implement middleware in a way that can be unit-tested:
  * Provide guard function `is_loopback(host: str) -> bool`
  * Test it directly with:
    * `127.0.0.1` → True
    * `::1` → True
    * `192.168.1.5` → False
    * `10.0.0.2` → False
* Add one integration-ish test by calling middleware with a crafted Request if possible; if not, document limitation and keep unit tests.

### 5) Documentation updates

Update `SPEC_MODULES/ui.md`:
* local-only enforcement behavior
* no trust in forwarded headers

Update `REPORTS/module_audit_ui.md`:
* risks reduced, remaining risks

## Acceptance criteria

* `pytest -q` passes
* `.\scripts\acceptance.ps1` passes
* Non-loopback client hosts are rejected with 403 + error page
* Dashboard shows local-only banner
* start_control_panel.ps1 forces loopback bind
* `CHANGELOG.md` + `REPORTS/AGENT_REPORT.md` updated with TASK-0030
* `CONTROL.md` reset to `CURRENT_TASK: NONE`

Revert middleware, banner, and script changes. Keep documentation updates for reference.
