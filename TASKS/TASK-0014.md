# TASK-0014 — UI Hardening: Dependency Split + Acceptance Artifacts

## Goal

Prevent dependency drift and make dashboard status accurate and durable.

## Scope

- requirements*.txt
- scripts/acceptance.ps1
- project_guardian/ui/app.py
- project_guardian/ui/templates/dashboard.html
- REPORTS/*
- tests/* (optional)
- REPORTS/AGENT_REPORT.md
- CHANGELOG.md
- TASKS/TASK-0014.md
- CONTROL.md

## Non-goals

- No UI redesign
- No auth

## Requirements

1. Dependency split:
   - Move UI deps to requirements-ui.txt (or requirements-dev.txt)
   - Keep core requirements.txt minimal and unchanged where possible
   - Update start_control_panel.ps1 to warn if UI deps missing

2. Acceptance artifacts:
   - acceptance.ps1 must write:
     - REPORTS/acceptance_last.json (timestamp, exit_code, pass/fail)
     - REPORTS/acceptance_last.log (full stdout/stderr)
   - UI reads these artifacts and displays:
     - last run time
     - pass/fail
     - link/view for last log (simple endpoint)

3. Safety:
   - Ensure logs redact any sensitive fields if they can appear (at least basic token/key patterns)

## Acceptance

- Run acceptance: artifacts written
- UI displays correct last-run info after restart
- No breaking changes to existing acceptance behavior

## Rollback

Revert requirements split and artifact writing.
