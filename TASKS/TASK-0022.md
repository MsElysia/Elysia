# TASK-0022 — Minimal Task Execution Engine (Whitelist Only)

## Goal

Add a **safe, deterministic task execution layer** to GuardianCore that executes only whitelisted task types. This completes the control loop without introducing arbitrary execution or prompt-injection risk.

## Scope (allowed files)

* `project_guardian/core.py`
* `CONTROL.md`
* `TASKS/*.md`
* `tests/test_task_execution.py` (new)
* `SPEC_MODULES/core.md` (update)
* `REPORTS/module_audit_core.md` (update)
* `TASKS/TASK-0022.md` (new)
* `CHANGELOG.md`
* `REPORTS/AGENT_REPORT.md`

## Non-goals

* No UI changes
* No arbitrary command execution
* No freeform markdown interpretation
* No network calls
* No new dependencies

## Required behavior

### 1) Task contract format (strict)

Each task file (`TASKS/TASK-XXXX.md`) may contain **exactly one directive line**:
* `TASK_TYPE: RUN_ACCEPTANCE`
* `TASK_TYPE: CLEAR_CURRENT_TASK`

Anything else:
* Missing directive
* Multiple directives
* Unknown task type

→ return structured error:
```json
{ "status": "error", "code": "TASK_TYPE_INVALID", "task_id": "TASK-XXXX" }
```

### 2) Task execution engine

Extend `GuardianCore.run_once()`:
If `status == "ready"`:
* Load task contract
* Extract `TASK_TYPE`
* Dispatch to execution handler

#### Allowed task types

**A) RUN_ACCEPTANCE**
* Invoke the existing acceptance runner **safely**
* Do not execute arbitrary commands
* Prefer calling the same internal path used by the UI (mockable)
* Return structured result:
```json
{
  "status": "ok",
  "task_id": "TASK-XXXX",
  "outcome": "acceptance_ran",
  "exit_code": <int>
}
```

**B) CLEAR_CURRENT_TASK**
* Atomically set `CONTROL.md` → `CURRENT_TASK: NONE`
* Return:
```json
{
  "status": "ok",
  "task_id": "TASK-XXXX",
  "outcome": "task_cleared"
}
```

### 3) Safety rules

* No `subprocess.run` except the known acceptance path
* No interpretation of markdown beyond the single directive
* No string error returns from `run_once()` — **dict only**

## Tests — `tests/test_task_execution.py`

Use `tmp_path` + monkeypatching.

### Required tests

1. Unknown task type → structured error
2. Missing TASK_TYPE → structured error
3. RUN_ACCEPTANCE task:
