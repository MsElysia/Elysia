# TASK-0018 — External Action Gateways Audit

## Goal

Prove the only external power surfaces behave correctly in allow/deny/review/replay paths, and do not leak sensitive context.

## Non-goals

- No changes to core TrustMatrix/ReviewQueue/ApprovalStore logic
- No UI changes
- No real external side effects (all tests must be mocked)

## Required behavioral tests

### A) WebReader (network)

1. **Deny path**
   - Force TrustMatrix decision = deny
   - Assert: raises `TrustDeniedError`, no network call attempted (patch requests/urllib)

2. **Review path**
   - Force decision = review
   - Assert: review request enqueued, raises `TrustReviewRequiredError` with request_id

3. **Approval replay**
   - Approve request_id with matching context
   - Call fetch with request_id
   - Assert: bypasses review and proceeds to "network call stub" (do not hit real internet; patch underlying call)

4. **Context safety**
   - Ensure stored context contains only domain/target (no full URL, no query string)

### B) FileWriter (filesystem)

1. **Deny path**: no write happens, raises TrustDeniedError
2. **Review path**: enqueues request + raises TrustReviewRequiredError
3. **Replay**: approved request_id allows write to a temp file only
4. **Mode restrictions**: Ensure modes are restricted to allowed set ("w/a/wb/ab" only)

### C) SubprocessRunner

1. **Deny path**: no subprocess run; raises TrustDeniedError
2. **Review path**: enqueues request + raises TrustReviewRequiredError
3. **Replay**: approved request_id allows subprocess.run to be called, but patch it (do not execute real commands)
4. **Timeout behavior**: confirm timeout raises TrustDeniedError with `SUBPROCESS_EXECUTION` constant

## Documentation

`SPEC_MODULES/gateways.md` must define:
- Each gateway's contract
- Action constants used
- What context is stored
- How replay is matched
- What is strictly forbidden (shell=True, arbitrary commands, uncontrolled writes)
