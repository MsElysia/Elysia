# TASK-0043 — End-to-End "No Side-Effects" Negative Tests (Deny/Review Must Not Write)

## Goal

Add end-to-end negative tests that prove the system creates **no side-effects** (no file changes, no backups, no logs/history artifacts beyond allowed queue entries) when actions are denied or require review.

This locks the invariants you've been hardening.

## Scope (allowed files)

* Tests (new):
  * `tests/test_no_side_effects_network.py`
  * `tests/test_no_side_effects_subprocess.py`
  * `tests/test_no_side_effects_mutation.py`
  * `tests/test_no_side_effects_task_engine.py` (optional if you want task-driven checks)
* Existing modules: **no code changes unless a test reveals a real violation**
* Docs:
  * `REPORTS/module_audit_core.md` (update if violations found/fixed)
  * `REPORTS/module_audit_gateways.md` (update if violations found/fixed)
  * `REPORTS/module_audit_mutation.md` (update if violations found/fixed)
* Task + logs:
  * `TASKS/TASK-0043.md`
  * `CHANGELOG.md`
  * `REPORTS/AGENT_REPORT.md`
  * `CONTROL.md`

### Non-goals

* No new features
* No refactors for style
* No relaxing invariants to "make tests pass"
* No network calls, no real subprocess execution

## Definitions

### "No side-effects" means:

* Target files unchanged
* No backup files created
* No mutation writes performed
* No subprocess started (Popen not called)
* No network performed (urllib called)
* Allowed exception: enqueueing a review request to ReviewQueue is a permitted side-effect for review decisions
* Allowed exception: writing acceptance artifacts/history only when RUN_ACCEPTANCE actually executes and is allowed (not part of deny/review tests)

## Test Setup Rules

* Use `tmp_path` as repo root.
* Instantiate components with injected repo_root where supported:
  * `MutationEngine(repo_root=tmp_path)`
  * `FileWriter(repo_root=tmp_path)`
* Ensure `REPORTS/` and `TASKS/` directories exist under tmp_path as needed.
* Monkeypatch any "project root" resolution in Core/UI if necessary so tests operate entirely in tmp_path.
* Mock external actions:
  * network: mock urllib / request layer
  * subprocess: mock subprocess.Popen/run
* Use deterministic fake TrustMatrix decisions (allow/deny/review) by stubbing `validate_trust_for_action`.

## Required Tests

### A) Network: SSRF denial must not call network

File: `tests/test_no_side_effects_network.py`

1. `test_internal_target_denied_no_network_call()`
   * Create WebReader with TrustMatrix stub returning allow (to ensure denial comes from SSRF floor)
   * Call `request_json("GET", "http://127.0.0.1/...", allow_internal=False)`
   * Assert:
     * raises TrustDeniedError reason_code TARGET_BLOCKED_INTERNAL
     * underlying urllib request function was NOT called

2. `test_internal_target_review_override_enqueues_only()`
   * allow_internal=True, TrustMatrix returns review
   * Assert:
     * raises TrustReviewRequiredError
     * ReviewQueue has 1 pending request
     * urllib not called

### B) Subprocess: deny/review must not launch process or write audit line

File: `tests/test_no_side_effects_subprocess.py`

1. `test_subprocess_review_no_launch_no_audit()`
   * TrustMatrix returns review
   * Call `run_command_background(...)`
   * Assert:
     * raises TrustReviewRequiredError
     * subprocess.Popen not called
     * `REPORTS/subprocess_background.jsonl` does not exist or is empty

2. `test_subprocess_deny_no_launch_no_audit()`
   * TrustMatrix returns deny
   * Assert same invariants

### C) Mutation: invalid path + review must create no backups/writes

File: `tests/test_no_side_effects_mutation.py`

1. `test_mutation_invalid_path_no_writes_no_backups()`
   * Prepare repo file `safe.txt` with content "A"
   * Create mutation payload touching:
     * `safe.txt` (valid)
     * `../evil.txt` (invalid)
   * Execute through the same path Core uses for APPLY_MUTATION preflight if feasible; otherwise call MutationEngine preflight/apply sequence used by core.
   * Assert:
     * `safe.txt` unchanged
     * no backup files exist anywhere under tmp_path (or at least none created for safe.txt)
     * mutation result is denied with PATH_TRAVERSAL_BLOCKED

2. `test_mutation_review_no_writes_no_backups()`
   * Payload with protected path + override true, TrustMatrix returns review
   * Assert:
     * raises/returns review-required outcome
     * file unchanged
     * no backups created
     * ReviewQueue has 1 request

### D) Task engine: APPLY_MUTATION review/deny must not write

File: `tests/test_no_side_effects_task_engine.py` (optional but recommended)

1. `test_task_apply_mutation_review_no_side_effects()`
   * Create CONTROL.md pointing to a task contract that runs APPLY_MUTATION
   * Provide mutation file in MUTATIONS/
   * TrustMatrix returns review
   * Call `GuardianCore.run_once()`
   * Assert:
     * returns needs_review
     * target file unchanged
     * no backups created

2. `test_task_apply_mutation_denied_no_side_effects()`
   * Invalid path or deny decision
   * Assert no writes/backups

(If Core is hard to root into tmp_path, skip this file and document limitation in the task report.)
